## 1. DST Operating Premise

DST is a controlled, semi-manual, LLM-assisted reasoning ecosystem. The traversal engine produces a recorded attempt under a kernel and intake. It does not own final authority.

A traversal may produce:
- candidate bridges;
- residues;
- obstruction maps;
- status labels;
- internal audits;
- jury-ready reports;
- proposed next prompts or preclusionary rules.

Claims remain conditional on declared scope, supplied reasoning, domain standards, jury assessment, and operator understanding.

## 2. Required Workflow

1. Advisor/operator construct or confirm formal intake.
2. Traversal engine runs in fresh project chat.
3. Traversal output returns to advisor/operator.
4. Advisor/operator prepare clean jury packet.
5. Jury runs in fresh external thread.
6. Jury returns structured report.
7. Advisor/operator accept, reject, narrow, or hold findings.
8. Accepted defect-preventing rules update this kernel.
9. Control ledger records the cycle.
10. Continuity state records the current project state if changed.
11. Next traversal runs in a new clean project chat.

No traversal may claim steps 3-11 occurred unless actually provided.

## 3. Source Role Separation

### 3.1 Kernel files
Binding procedural rules. Mutable only after accepted jury/operator/advisor findings.

### 3.2 Canon files
Heuristic and translational substrate: worldview, framing, preferred vocabulary, interpretive models, collaborator writings, prior finalized continuity records. Canon is not proof authority.

### 3.3 Record files
Historical logs of prompts, traversals, jury reports, operator/advisor decisions, and continuity states. Records preserve context but do not automatically certify their claims.

### 3.4 Intake
Problem-specific formal base, targets, nullification conditions, domain standards, proof standards, constraints, and tolerance. The intake determines the active traversal field.

### 3.5 Domain standards
Outside domain standards apply as declared or implied by the intake. In mathematics: proof, implication, counterexample, equivalence, topology, citation discipline, and scope must meet ordinary mathematical standards.

## 12. Departure, Suspension, and Reengagement

O2. Disciplined Departure is not closure, finality, proof of completeness, or a claim that no criticism is possible. It is tolerance-bounded disengagement from further refinement when no blocking defect remains and marginal refinement adds more verbal mass than epistemic gain.

O2 conditions:
1. No current blocking defect.
2. No genuinely new material defect class.
3. Remaining criticisms are non-halting refinements or relabelings.
4. Diminishing marginal returns shown.
5. Further iteration risks more verbal mass/audit complexity than correctness gain.

O2a. Reengagement certificate synchronization:
Any recommendation of Disciplined Departure, suspension, route exhaustion, or OPEN-FORMAL RESIDUE status must synchronize O2, provenance, bridge catalogue, and status labels by naming:

2. reason for departure/suspension;
3. preserved gains;
4. live bridge directions or reason none remain;

## 13. Toeplitz/Inscribed Square Generalized Preclusionary Rules

These came from prior Toeplitz traversal history but are generalized enough to remain kernel law for any relevant configuration-space traversal.

G13. Dimension-matched forcing gate: before using a degree/index/obstruction forcing theorem, verify domain dimension, codomain dimension, representation dimension, and scalar equation count separately.

G14. Relative degree gate: a relative degree argument requires a compactified pair, nonvanishing boundary section after any required deletion, coefficient system, and target cohomology group.

G17. Before treating relative equivariant cohomology as merely open, test whether group action on compactified pair is free; if free, pass to quotient and check orientability.

G18. Audit orientation sign of cyclic permutation action on simplex coordinates. If quotient is nonorientable over untwisted Z, do not assume top relative cohomology survives.

G21. Boundary-zero audit: classify zeros on every boundary stratum, including wraparound faces and quotient collapse strata. Do not compress a positive-dimensional collapse locus into a single point.

G22. No Taylor/leading-order expansion for merely continuous curves. Smooth/local expansions require stated regularity and correct local parameter model.

G23. Boundary-normalized maps near collapse require correct deletion/tubular-neighborhood construction and all new boundary components.

G24. Unordered theorem vs ordered component gate: existence of an unordered geometric object does not imply existence in a specific labeled/cyclic component. A separate theorem or labeling proof preserving required structure is needed.

G25. Labeled vs unlabeled transfer: any transfer from geometric/unlabeled theorem to labeled configuration claim must state labeling map, cyclic order convention, diagonal pairing, and membership in selected component.

G26. Gap-simplex collapse consistency: full collapse must be described by actual simplex vertices/faces, not impossible all-gap-zero language.

G27. Cyclic sign-reversal premise: sign-reversal gives opposite signs only after verifying a non-square point in the genuine cyclic rectangle locus, or verifying that all such points are squares, which already closes the target.

G28. External theorem output type: classify whether theorem gives existence of one object, connected set, path, positive-measure family, all aspect ratios, or a trichotomy. Do not transfer output type.

G29. Vaughan/Meyerson scope: rectangle existence alone is unordered unless mechanism/order-compatible refinement is verified.

G30. Before declaring G24 not cleared in Toeplitz-style rectangle routes, assess known graceful/order-compatible rectangle theorems relevant to the exact component.

G31. Connected vs path-connected distinction: a connected set does not automatically supply an IVT path unless path-connectedness or suitable continuum argument for the specific function is established.

G32. CCC checklist: to replace global path-connectedness, explicitly prove the weaker component-connectivity condition needed: x connected/path-connected to sigma*x, or some connected component contains opposite signs, or x already square.
